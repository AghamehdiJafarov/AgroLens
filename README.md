# AgroLens — оценка качества плодоовощной продукции по фото

Первый из двух сайтов проекта. Инструмент, который по фотографии плода или партии
считает состав цвета, находит тёмные зоны (вероятные дефекты) и выставляет сорт
A / B / C по стандарту, который задаёт сам пользователь под своего покупателя.

Интерфейс на трёх языках: русский, английский, азербайджанский.

## Что это делает честно

Всё, что показывает инструмент, — результат реальных вычислений над пикселями снимка:

- **Состав цвета** — доля красного, оранжевого, жёлтого, зелёного и тёмного/бурого
  на плоде. Считается по среднему оттенку каждого пикселя (HSV). Это точная арифметика.
- **Тёмные зоны** — доля площади плода, которая заметно темнее его основного тона.
  Сигнал пятен, гнили, побитости. Порог чувствительности настраивается.
- **Калибр** — насколько плод занимает кадр. Это **относительная** величина.
  Абсолютные миллиметры требуют эталона в кадре (например, монеты).
- **Сорт A/B/C** — выставляется строго по порогам, которые ввёл пользователь.
  Прозрачно и воспроизводимо: одинаковое фото + одинаковые пороги = одинаковый сорт.

## Чего он НЕ делает (и не притворяется, что делает)

- Не отличает одну болезнь от другой — для этого нужна нейросеть, обученная на
  тысячах местных снимков. Здесь её нет.
- Не даёт миллиметры без эталона в кадре.
- Не заявляет 99% точности. Реалистичная планка обученной модели в поле — около 90%.
- Не заменяет лабораторию (микотоксины, внутренняя зрелость).

Ровно эти границы честно перечислены прямо на сайте, в секции «Честно о точности».

## Приватность

Фотография **не покидает устройство**. Весь анализ выполняется в браузере
на `<canvas>`. Нет сервера, нет загрузки фото куда-либо, нет API-ключа.

## Как запустить

Нужен Node.js (версия 18 или новее).

```bash
npm install     # поставить зависимости
npm run dev     # запустить локально → откроется http://localhost:5173
```

Для сборки готового сайта:

```bash
npm run build   # результат в папке dist/ — её можно залить на любой хостинг
npm run preview # посмотреть собранный сайт локально
```

## Структура

```
src/
  main.jsx      — точка входа, маршруты: / (главная) и /app (инструмент)
  lang.jsx      — общий переключатель языка RU/EN/AZ + весь словарь переводов
  Landing.jsx   — главная страница (маркетинг)
  App.jsx       — рабочий инструмент: стандарт покупателя, загрузка фото, результат
  analyze.js    — движок анализа изображения (вся честная математика здесь)
  index.css     — шрифты и базовые стили
```

Вся логика анализа изолирована в одном файле `analyze.js` — его можно прочитать
и проверить, что никаких выдуманных чисел там нет.

## Технологии

React 18 · Vite · Tailwind CSS · Recharts (графики) · lucide-react (иконки).
Тот же стек, что и во втором сайте проекта, — чтобы оба выглядели как одна система.

## Дальнейшее развитие

По мере того как через инструмент проходят фотографии, они складываются в
размеченную базу под местные культуры (Губа, Хачмаз, Гёйчай, Ленкорань).
Из этой базы позже обучается модель, которая добавит распознавание болезней —
это путь наращивания, а не переписывания. Так же росли базы у промышленных
систем: с каждой инспекцией.
